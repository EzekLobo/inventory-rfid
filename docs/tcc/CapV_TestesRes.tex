\chapter{Conclusão}

Este trabalho apresentou o desenvolvimento de um protótipo funcional de sistema web para inventário patrimonial baseado em RFID, aplicado ao contexto do Colegiado de Ciência da Computação da UESC. A proposta partiu da identificação de limitações associadas ao controle patrimonial manual, como dependência de conferência individual, possibilidade de inconsistências e dificuldade de manter o inventário físico alinhado aos registros lógicos.

O objetivo geral foi atendido por meio da implementação de uma aplicação web com API REST, processamento de eventos RFID, persistência de dados e interface de acompanhamento operacional. O sistema permite cadastrar locais, leitores e itens patrimoniais, registrar leituras, atualizar localização física, executar auditorias, gerar logs e apontar inconsistências. Dessa forma, o protótipo demonstra uma base computacional viável para apoiar processos de inventário e rastreabilidade patrimonial.

Os objetivos específicos também foram contemplados. O levantamento de requisitos foi consolidado nas tabelas de requisitos funcionais e não funcionais, a arquitetura modular foi descrita com separação entre frontend, backend, API, banco de dados e dispositivos de leitura, e o protótipo web foi implementado com telas de consulta, cadastro, auditoria, inconsistências e log operacional. Além disso, a integração com leitor RFID de proximidade permitiu validar fisicamente o fluxo básico de leitura e envio de tags à API, enquanto as verificações funcionais de software demonstraram o comportamento previsto para eventos representativos de sensores ou leitores em rede.

Um aspecto central da solução é sua organização modular. A aplicação foi concebida para receber eventos de diferentes dispositivos, desde que estes se comuniquem com a API no formato esperado. Na validação realizada, utilizou-se um leitor RFID de proximidade de baixo custo, escolha adequada à disponibilidade de equipamentos e ao objetivo de comprovar o fluxo lógico da solução. Essa decisão não esgota as possibilidades de implantação, mas demonstra que o núcleo computacional do sistema pode ser preservado em cenários com dispositivos de captura distintos.

As limitações do trabalho devem ser interpretadas dentro desse escopo. A validação não avaliou desempenho de antenas RFID de longo alcance, leitura simultânea em massa, interferência eletromagnética, sensor infravermelho físico ou comportamento em ambiente institucional de produção. Também não se deve generalizar métricas de alcance ou acurácia sem ensaios específicos. O fluxo de acionamento por sensor foi preservado na arquitetura e verificado funcionalmente por software, mas sua validação física permanece como etapa futura. Assim, os resultados representam validação funcional do software e do pipeline de processamento RFID, e não validação completa de uma infraestrutura física definitiva.

Apesar dessas limitações, o protótipo contribui ao demonstrar que é possível integrar cadastro patrimonial, eventos RFID, auditoria, histórico operacional e tratamento de inconsistências em uma aplicação web. Como trabalhos futuros, destacam-se a realização de testes com antenas RFID de maior alcance, a integração física de sensores de presença, a ampliação da base experimental com maior quantidade de ativos, a substituição do banco local por solução mais robusta, o aprimoramento da autenticação e a geração de relatórios formais de inventário e auditoria.

Conclui-se, portanto, que a solução desenvolvida atende ao propósito de validar uma arquitetura de software para inventário patrimonial baseado em RFID, oferecendo uma base funcional e extensível para futuras implantações e experimentos em ambientes acadêmicos. A principal contribuição está na entrega de um protótipo integrado, capaz de conectar leitura RFID, API, processamento de regras, persistência, auditoria e rastreabilidade operacional sem extrapolar os limites da validação física realizada.
