\begin{resumo}
O controle patrimonial em ambientes acadêmicos pode ser dificultado pela movimentação de equipamentos, pela dependência de conferência manual e pela ocorrência de divergências entre registros administrativos e localização física dos bens. Este trabalho apresenta o desenvolvimento de um protótipo funcional de sistema web para inventário patrimonial baseado em Identificação por Radiofrequência (RFID), aplicado ao contexto do Colegiado de Ciência da Computação da Universidade Estadual de Santa Cruz. A solução foi implementada com frontend web, backend com API REST, banco de dados e processamento de eventos RFID, contemplando cadastros de locais, leitores e itens patrimoniais, além de auditoria, histórico operacional e registro de inconsistências. A validação foi conduzida em cenário controlado com leitor RFID de proximidade de baixo custo e uma tag, complementada por verificações funcionais de software para o fluxo previsto com sensores ou leitores em rede. Os resultados indicam que o protótipo atende ao objetivo de validar o pipeline computacional da solução, recebendo eventos de leitura, processando tags, atualizando informações do inventário, registrando histórico e identificando divergências. A avaliação, entretanto, não mede alcance, acurácia, leitura simultânea em massa ou desempenho de antenas RFID de longo alcance, ficando esses aspectos como continuidade experimental.

 \textbf{Palavras-chave}: Inventário patrimonial. RFID. Sistema web. Auditoria patrimonial. Rastreabilidade.
\end{resumo}

% resumo em inglês
\begin{resumo}[Abstract]
 \begin{otherlanguage*}{english}
Asset inventory control in academic environments can be hindered by equipment movement, reliance on manual checking, and inconsistencies between administrative records and the physical location of assets. This work presents the development of a functional web system prototype for asset inventory based on Radio-Frequency Identification (RFID), applied to the context of the Computer Science Department at the State University of Santa Cruz. The solution was implemented with a web frontend, a REST API backend, a database, and RFID event processing, covering the registration of locations, readers, and assets, as well as auditing, operational history, and inconsistency records. The validation was conducted in a controlled scenario using a low-cost short-range RFID reader and a single tag, complemented by functional software checks for the proposed flow with networked sensors or readers. The results indicate that the prototype meets the objective of validating the computational pipeline of the solution by receiving reading events, processing tags, updating inventory information, recording history, and identifying divergences. However, the evaluation does not measure range, accuracy, bulk simultaneous reading, or the performance of long-range RFID antennas, leaving these aspects for future experimental work.

   \vspace{\onelineskip}
 
   \noindent 
   \textbf{Keywords}: Asset inventory. RFID. Web system. Asset auditing. Traceability.
 \end{otherlanguage*}
\end{resumo}
